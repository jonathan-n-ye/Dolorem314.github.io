Object Oriented Programming

Java is uses Objects and Classes. They have an identity (name), a state (properties), and a behaviour (actions it can do).

Classes act as blueprints for Objects and Objects act as the actual things made with these blueprints. Classes can be already predefined within Java or be written by programmers.

To create an Object, the following code is used:

ClassType classname; //Initialization
classname = new ClassType("Hey"); //Defining the new class
//The constructor is called while defining, passing String "Hey"

class ClassType{
	
	//Attributes
	String variable1;
	int variable2;
	
	ClassType(String v1){ //Constructor with the parameter of a String
		variable1 = v1;
		variable2 = 10;
	}
	
	void method1(){
		System.out.println(variable1)'
	}
	
}

Classes have variables and methods:

Types of methods:

  1. Static Method: Also known as class methods, they can be run without having an Object.
  1. Constructor: It constructs the object. Java will automatically create a blank constructor if no constructor is defined.
  1. Instance Method: Instance methods are methods that can be used by an Object. It does things within the Object.
    1. get() methods: methods used to get the value of a certain variable in the class
    1. set() methods: methods used to set a variable to a certain value in a class

Methods and Variables also have access levels. They can be:

  1. private - only methods inside the class can access them
  1. protected - only its family can access them
  1. public - anything can access them

Inheritance

Classes can Inherit other Classes. When inheriting, there is always a superclass and a subclass. The subclass inherits the superclass. You should always be able to say that the subclass is a superclass. For example, a dog (subclass) is an animal (superclass).

To inherit another class, the following code is used:

public class Animal{}
public class Dog extends Animal{} //inherits the animal class

Abstract Classes

Abstract classes are used in polymorphism. They are classes that also contain abstract methods.
